Описание программы

CORELAP (Computerized Relationship Layout Planning) - это программа для автоматизированной компоновки производственных подразделений (цехов, участков) в здании с заданными параметрами. Программа использует алгоритм CORELAP, который учитывает грузопотоки между подразделениями и стремится минимизировать суммарный грузопоток (т×м/год) за счёт размещения сильно связанных подразделений рядом.

Функциональные возможности:
1. Автоматическое размещение подразделений на основе матрицы грузопотоков
2. Ранжирование подразделений по важности (5 категорий: от "очень важный" до "совсем неважный")
3. Ручные перестановки подразделений после автоматического размещения
4. Визуализация компоновки в HTML-формате с цветовой индикацией категорий
5. Сохранение результатов в HTML-файл для дальнейшего анализа
6. Перезапуск алгоритма с теми же данными для поиска лучшего варианта

Требования к системе:
- Операционная система: Windows / Linux / macOS
- Компилятор: C++17 или выше
- Поддержка кириллицы (setlocale LC_ALL "Russian")

Установка и компиляция:
- Windows (Visual Studio)
cl /EHsc corelap.cpp
- Linux / macOS (g++)
g++ -std=c++17 -o corelap corelap.cpp

Формат входных данных:
1. Параметры здания
- Ширина пролёта (м) - определяет вертикальный размер подразделений (обычно 12-18 м)
- Количество пролётов - количество параллельных рядов для размещения (1-20)
- Длина здания (м) - максимальная длина подразделения (1-500 м)
2. Подразделения
Формат ввода: `Название, Площадь`
3. Матрица грузопотоков
- Ввод построчно для каждого подразделения
- Значения грузопотоков по диагонали одинаковых подразделений не вводить
- Значения - грузопоток в тоннах в год (т/год)
- Потоки встречных направлений автоматически суммируются
- Для отсутствия потока используйте `0` или `-`

Пример ввода информации о подразделениях:
Сборочный цех, 600
Механический цех, 450
Склад готовой продукции, 300

Пример ввода матрицы грузопотоков для трёх подразделений (грузопотоки между одним подразделением не включены):
150 80
0 200
0 0

Основные этапы работы:
1. Ввод исходных данных
   - Параметры здания
   - Список подразделений с площадями
   - Матрица грузопотоков
2. Автоматическое размещение
   - Программа выполняет расчёт и выводит результаты
   - Отображается детализация связей и суммарный грузопоток
3. Ручные перестановки (опционально)
   - Обмен двух подразделений
   - Перемещение подразделения на другое место
   - Удаление подразделения
4. Сохранение результатов
   - HTML-файл с визуализацией компоновки

Ограничения:
- Максимальное количество подразделений: 100
- Максимальная площадь подразделения: 10000 м²
- Минимальная длина подразделения: 1 м
- Подразделение всегда занимает целую ширину пролёта

Файлы, создаваемые программой:
- `layout_XX_YY.html` - HTML-файл с визуализацией компоновки
  - `XX` - номер запуска программы
  - `YY` - номер сохранения в текущем сеансе

Обработка ошибок:
- При вводе некорректных данных программа запрашивает повторный ввод
- При невозможности разместить подразделение выводится соответствующее сообщение
- Автоматическое восстановление при сбоях

Программа сгенерирована нейросетью и разработана в учебных целях для демонстрации работы алгоритма CORELAP.
Программа использует кириллицу, убедитесь, что консоль поддерживает кодировку UTF-8 или CP1251.
